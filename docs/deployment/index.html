WebGL Deployment

Build Configuration

  1. Open Build Settings

    Navigate to File > Build Settings in Unity Editor.

  2. Configure WebGL Settings

    Ensure the following settings are configured:

    • Platform: Select WebGL tab
    • Scenes: Only LoadAll scene should be checked
    • Code Optimization:
      • Use Shorter Build Time for testing
      • Use Disk Size with LTO for production deployment

Testing Build

  1. Development Testing

    Set Code Optimization to Shorter Build Time, then go to File > Build and Run to test in a simulated online environment.

  2. Production Build

    After testing, change Code Optimization to Disk Size with LTO and build the Unity project to Builds/WebGL/ folder.

Web Integration

  1. Deploy to Web Application

    Copy all built files from the Unity WebGL build output to your web application:

    Source: UnityProject/Builds/WebGL/
Destination: mikomiko-next-app/public/WebGL/

  2. Test Deployment

    Run the website locally to verify your changes:

    npm run build
  • Use Disk Size with LTO for production builds to minimize file size
  • Test with Shorter Build Time during development for faster iteration

iOS Deployment

Unity Build Setup

  1. Configure iOS Platform

    • Go to File > Build Settings
    • Switch Platform from WebGL to iOS
    • Build the Unity project and save to Builds/iOS/ folder

Xcode Integration

  1. Open Xcode Project

    Launch mikodemo.xcodeproj in Xcode.

  2. Unity Framework Setup

    Check Unity-iPhone presence:

    • If Unity-iPhone is expandable in the project navigator, skip to step 4
    • If not expandable, remove the reference and proceed to step 4

  3. Import Unity Project

    Drag Unity-iPhone.xcodeproj from Builds/iOS/ folder into your Xcode project.

    Configure frameworks:

    • Re-import the UnityFramework.framework library in Xcode
    • Select the Data folder and check the Target Membership box next to UnityFramework

  4. Configure Native Plugins

    Navigate to Unity-iPhone > Libraries > Plugins > iOS:

    Set Target Membership to Public for:

    • NativeCallProxy.h
    • NativeState.h

    Under Target Membership, ensure UnityFramework is set to Public.

  5. Update UnityFramework Header

    Add this include statement to the top of the UnityFramework.h file.

    #include "NativeCallProxy.h"

Common Issues

  • Missing scenes: Ensure only LoadAll scene is included in build
  • Too little Memory: Sometimes the iOS Build fails because there is too little free memory. Make sure to clean memory inside XCode once in a while
  • Animations not Working: If we are still using the animation method for loading poses, please go Canvas > SideMenu > Scroll View > Viewport > Content inside the scene, right click on the component Auto Fill Pose(Script) and click on generate pose libraries.

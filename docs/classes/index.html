Class Naming Conventions

  • Controllers: Classes ending in Controller handle user input and system state
  • Managers: Classes ending in Manager coordinate multiple systems
  • Utilities: Classes ending in Util or Helper provide utility functions
  • Data: Classes ending in Data represent data structures and containers

Development Guidelines

Adding New Classes

  1. Documentation: Create comprehensive documentation for any new class
  2. Naming: Follow established naming conventions
  3. Dependencies: Clearly document class dependencies and requirements
  4. Examples: Provide usage examples and integration guides

Modifying Existing Classes

  1. Backward Compatibility: Maintain API compatibility when possible
  2. Documentation Updates: Update documentation to reflect changes
  3. Testing: Verify changes don’t break existing functionality
  4. Version Notes: Document breaking changes and migration paths

Getting Help

If you need assistance with any specific class:

  1. Check the individual class documentation in the respective folders
  2. Review the Topics Documentation for high-level concept explanations
  3. Consult the Project Overview for system architecture
  4. Contact the development team for advanced support

For conceptual information about these systems, see the Topics Documentation.
